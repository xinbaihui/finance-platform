# finance-platform

`finance-platform` 是一个面向个人理财场景的智能产品原型，目标是帮助用户更清晰地管理收入、支出、资产和年度储蓄目标，并通过 AI 对财务数据进行解释与辅助分析。

产品当前由两个核心部分组成：

- `理财副驾驶`：面向普通用户的移动端产品，用于录入个人财务信息、查看年度分析结果，并通过聊天方式获取财务建议。
- `管理后台`：面向运营或演示场景的 Web 管理端，用于查看用户列表、用户财务画像和关键指标，方便进行管理、排查与产品演示。

## 产品说明

我做“理财副驾驶”，是想打造一个让用户能够更快了解自身财务状况、及时发现问题并获得建议的智能理财助手。和传统记账产品不同，它不要求用户把每一笔花费都非常细致地记录下来，而是通过一次性录入资产、收支和年度目标等关键信息，帮助用户快速形成整体财务画像。基于这些数据，系统可以进一步分析用户当前的财务健康状况、判断年度目标是否有达成可能，并在发现支出压力或目标差距时，及时给出更有针对性的建议。

这个产品的核心价值主要体现在以下几个方面：

- 降低理财记录门槛，不要求用户长期、逐笔地精细记账。
- 把分散的财务数据整合为统一视图，帮助用户快速看清自己的整体财务状况。
- 将“记录数据”与“解释数据”结合起来，不只是展示数字，还能解释趋势、差距和潜在问题。
- 通过 AI 对年度目标、储蓄达成情况和支出结构进行辅助分析，并在发现问题时及时给出建议。

从用户视角看，这个产品希望回答几个实际问题：

- 我今年挣了多少、花了多少、存下了多少？
- 当前支出结构是否健康，是否存在超支风险？
- 我的年度储蓄目标还差多少，接下来应该优先调整哪些支出？
- 如果收入增加或支出减少，全年结果会发生什么变化？

## 项目技术栈

本项目采用 Monorepo 组织方式，覆盖移动端、Web 管理端、后端服务和数据库建模，整体技术栈如下：

### 前端技术栈

移动端：

- `React Native`
- `Expo`
- `expo-router`
- `TypeScript`

说明：

- 使用 Expo 进行多端开发与调试，提升开发效率。
- 使用 `expo-router` 管理页面路由，便于组织 Chat、分析、我的等页面结构。
- 使用 `react-native-svg` 绘制图表，用于呈现支出结构等可视化信息。

管理后台：

- `React`
- `Vite`
- `TypeScript`

说明：

- 管理后台以轻量化 Web 应用形式实现，适合快速构建演示型后台系统。
- 使用 React 组件化开发，便于后续扩展用户管理、配置管理和权限管理等后台能力。

### 后端技术栈

- `Python`
- `Flask`
- `SQLAlchemy`
- `Pydantic / pydantic-settings`

说明：

- 后端负责财务数据聚合、年度分析、上下文构建、聊天接口封装和后台管理接口提供。
- 使用 Pydantic 进行配置管理和数据校验，保证接口输入输出结构清晰可控。

### AI 与数据能力

- `OpenAI API`
- `Codex CLI`

说明：

- 聊天能力支持两种 provider：`openai` 和 `codex`。
- 在本地演示模式下，可以通过 `Codex CLI` 直接生成回答；在标准 API 模式下，可以通过 OpenAI 模型完成回复生成。
- 后端会根据用户问题，动态拼装财务上下文，例如财务快照、储蓄目标分析、支出结构分析、全年预测和建议输入，再传递给模型生成结果。

### 数据库与工程化

- `PostgreSQL`
- `Prisma`
- `pnpm workspace`
- `Nx`

说明：

- Prisma 用于数据库模型定义，当前已建模用户、财务画像、年度计划、交易记录、聊天会话等核心数据结构。
- 使用 `pnpm workspace` 管理多应用依赖。
- 使用 `Nx` 统一组织移动端、后台端和服务端任务，便于在 Monorepo 下进行开发与维护。
